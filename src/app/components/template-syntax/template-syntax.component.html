Template Syntax

How is the script tag treated in and HTML template?

SCRIPT tags are simply ignored to prevent safety risks of encripting scripts

How are different the HTML attributes than the DOM properties?

Attributes are defined by HTML. Properties are defined by the DOM (Document Object Model).

Attributes initialize DOM properties and then they are done. Property values can change; attribute values can't.

The HTML attribute and the DOM property are not the same thing, even when they have the same name.

Template binding works with properties and events, not attributes. In the world of Angular, the only role of attributes is to initialize element and directive state. When you write a data binding, you're dealing exclusively with properties and events of the target object. HTML attributes effectively disappear.

What is ONE-WAY binding?

It means that you cannot use property binding to pull values out of the target element. You can't bind to a property of the target element to read it. You can only set it.

Are property binding and interpolation the same?

Interpolation uses double curly braces to delimit expressions into HTML. Property binding evaluates an expressions and passes the value to the component property.

When binding data values as strings both can be used exactly for the same. The implementation though is defferent: 

{{'<'}}p>{{'<'}}img src="{{ '{{' }}heroImageUrl}}"> is the {{'<'}}i>interpolated{{'<'}}/i> image.{{'<'}}/p>
{{'<'}}p>{{'<'}}img [src]="heroImageUrl"> is the {{'<'}}i>property bound{{'<'}}/i> image.{{'<'}}/p>

{{'<'}}p>{{'<'}}span>"{{ '{{' }}title}}" is the {{'<'}}i>interpolated{{'<'}}/i> title.{{'<'}}/span>{{'<'}}/p>
{{'<'}}p>"{{'<'}}span [innerHTML]="title">{{'<'}}/span>" is the {{'<'}}i>property bound{{'<'}}/i> title.{{'<'}}/p>

When do we use Attribute Binding?

This is used to set a value to an attribute directly. This is the only exception for targeting a property of the element. We must use attribute binding when there is no property to bind.

The use of attribute binding is needed to cases like ARIA, colspan, svg for example since they are pure attributes. 

{{'<'}}button [attr.aria-label]="actionName">{{actionName}} with Aria{{'<'}}/button>
{{'<'}}tr>{{'<'}}td [attr.colspan]="1 + 1">One-Two{{'<'}}/td>{{'<'}}/tr>

When do we use Class Binding and when ngClass?

When adding or removing classes from the class attribute. Class binding allows you to bind all-or-nothing class names as well as single classes. NgClass directive is used when multiple class names have to be managed at the same time. 

{{'<'}}!-- reset/override all class names with a binding  -->
{{'<'}}div class="bad curly special" [class]="badCurly">Bad curly{{'<'}}/div>
{{'<'}}!-- toggle the "special" class on/off with a property -->
{{'<'}}div [class.special]="isSpecial">The class binding is special{{'<'}}/div>

When do we use Style Binding and when ngStyle?

To set inline styles we can use style binding. This is fine for single styles but the directive NgStyle is preferred when several inline styles are to be set. 

{{'<'}}!-- The style property cna be written with dash separator or camelCase notation  -->
{{'<'}}button [style.color]="isSpecial ? 'red': 'green'">Red{{'<'}}/button>
{{'<'}}button [style.background-color]="canSave ? 'cyan': 'grey'" >Save{{'<'}}/button>
{{'<'}}button [style.backgroundColor]="canSave ? 'cyan': 'grey'" >Save{{'<'}}/button>

Why is it said that "The two-way binding syntax is really just syntactic sugar for a property binding and an event binding"?

Angular "desugars" 2 way binding to property and event binding. 

{{'<'}}!-- Angular assigns the $event value to the fontSizePx variable.  -->
{{'<'}}app-sizer [(size)]="fontSizePx">{{'<'}}/app-sizer>
{{'<'}}app-sizer [size]="fontSizePx" (sizeChange)="fontSizePx=$event">{{'<'}}/app-sizer>

Are NgClass, NgModel, NgStyle attribute directives or structural directives?

They are examples of attribute directives. Attribute directives are meant to modify the behavior of HTML elements, attributes, properties or components.

  • NgClass - add and remove a set of CSS classes
  • NgStyle - add and remove a set of HTML styles
  • NgModel - two-way data binding to an HTML form element

What is Content Projection?

Known in AngularJS as transclusion it allows configurable content to be displayed in specific parts of a component. It uses the directive ng-content. There are several ways to implement content projection in Angular: by class, by selector. The component anguide-question in this guide is an example of content projection.
